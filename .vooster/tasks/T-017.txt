# 메인 Hero 영역 및 AI 챗봇 UI 우선 리디자인

**Task ID:** T-017
**Status:** DONE
**Importance:** MUST
**Complexity:** 7/10
**Urgency:** 9/10
**Dependencies:** T-004, T-016

## Description

### 설명
사용자의 첫인상과 브랜드 신뢰도에 가장 큰 영향을 미치는 메인 페이지의 Hero 영역과 AI 챗봇 인터페이스에 새로운 디자인 시스템을 우선적으로 적용합니다. 3D, 애니메이션 등 동적인 요소를 추가하여 사용자의 시선을 사로잡고, 챗봇 접근성을 강화하여 즉각적인 상담으로 이어지도록 유도합니다.

### 구현 상세
1.  **Hero 영역 컴포넌트 개발**: 메인 페이지 최상단에 위치할 Hero 영역 컴포넌트를 생성합니다. 고해상도 이미지나 배경 비디오를 사용하여 전문적인 느낌을 강조합니다.
2.  **동적 효과 추가**: `Framer Motion` 라이브러리를 사용하여 스크롤에 따른 패럴랙스 효과, 텍스트/이미지의 페이드인 애니메이션 등을 구현하여 생동감을 더합니다. 3D 모델이 필요할 경우 `three.js`와 `@react-three/fiber`를 활용하여 통합합니다.
3.  **AI 챗봇 진입점(CTA) 개선**: AI 챗봇으로 연결되는 CTA(Call To Action) 버튼과 플로팅 아이콘을 새 디자인 가이드에 맞춰 눈에 띄게 재디자인합니다. 사용자의 시선이 자연스럽게 머물도록 배치하고, 마이크로 인터랙션을 추가합니다.
4.  **AI 챗봇 UI 리디자인**: 기존 AI 챗봇(T-004)의 대화창, 메시지 버블, 입력창, 버튼 등의 UI를 `T-016`에서 정의한 새로운 공통 컴포넌트와 스타일로 교체하여 전체 페이지와의 디자인 통일성을 확보합니다.

### 테스트 전략
1.  **반응형 레이아웃 테스트**: 다양한 디바이스(모바일, 태블릿, 데스크탑)와 브라우저(Chrome, Safari, Edge)에서 Hero 영역의 레이아웃이 깨지거나 요소가 겹치지 않는지 확인합니다.
2.  **애니메이션 성능 테스트**: 브라우저 개발자 도구의 Performance 탭을 사용하여 애니메이션이 60fps를 유지하는지 확인하고, CPU 사용량이 과도하게 높아지지 않는지 점검합니다.
3.  **사용성 테스트**: 새로운 Hero 영역의 CTA 버튼과 챗봇 플로팅 아이콘이 명확하게 인지되고 쉽게 클릭할 수 있는지, 챗봇 UI가 직관적인지 실제 사용자를 대상으로 테스트를 진행하거나 동료 리뷰를 받습니다.
4.  **A/B 테스트**: 가능하다면 기존 Hero 영역과 새로운 Hero 영역의 전환율(예: 챗봇 시작률)을 비교하는 A/B 테스트를 설정하여 디자인 개선 효과를 데이터로 검증합니다.
