# 예약/상담 퍼널 UI/UX 리디자인 및 단계별 폼 개선

**Task ID:** T-018
**Status:** DONE
**Importance:** MUST
**Complexity:** 7/10
**Urgency:** 8/10
**Dependencies:** T-005, T-006, T-016

## Description

### 설명
사용자 이탈이 가장 많이 발생하는 예약 및 상담 신청 퍼널의 UI/UX를 전면 개선합니다. 기존의 긴 폼을 Typebot과 같이 대화형으로 질문을 던지는 단계별 폼(Multi-step Form)으로 변경하여 사용자 피로도를 줄이고, 새 디자인 가이드를 적용하여 시각적 완성도와 신뢰도를 높입니다.

### 구현 상세
1.  **단계별 폼 컴포넌트 구현**: 사용자의 입력을 여러 단계(예: 1단계-개인정보, 2단계-희망 날짜, 3단계-증상 입력)로 나누어 진행하는 `MultiStepForm` 컴포넌트를 구현합니다. 각 단계의 상태는 `useState`나 `useReducer`로 관리합니다.
2.  **대화형 UI 적용**: 각 단계에서 하나의 질문에만 집중할 수 있도록 UI를 구성하고, 단계 전환 시 `Framer Motion`을 사용하여 부드러운 슬라이드 또는 페이드 효과를 적용하여 시각적 흐름을 개선합니다.
3.  **캘린더 UI 커스터마이징**: `react-day-picker`와 같은 캘린더 라이브러리를 사용하여 예약 가능 날짜를 표시하고, `T-016`의 디자인 시스템에 맞게 색상, 폰트, 레이아웃을 커스터마이징합니다.
4.  **유효성 검사 및 피드백 강화**: `react-hook-form`과 `zod`를 사용하여 각 입력 필드에 대한 실시간 유효성 검사를 구현합니다. 오류 발생 시 명확하고 친절한 에러 메시지를 즉시 표시하고, 예약 성공/실패 시 피드백 UI(토스트, 모달 등)를 강화합니다.

### 테스트 전략
1.  **퍼널 완료 테스트 (End-to-End)**: 사용자가 첫 단계부터 마지막 단계까지 모든 정보를 입력하고 예약을 완료하는 전체 시나리오를 테스트합니다. 최종적으로 데이터가 Supabase DB(T-002)에 정확하게 저장되는지 확인합니다.
2.  **유효성 검사 케이스 테스트**: 필수 입력 필드를 비워두거나, 이메일/전화번호 형식을 틀리게 입력하는 등 다양한 예외 케이스를 테스트하여 에러 메시지가 정상적으로 표시되는지 검증합니다.
3.  **상태 유지 테스트**: 사용자가 폼 작성 중 페이지를 새로고침하거나 뒤로 가기/앞으로 가기를 했을 때, 이전에 입력한 정보가 유지되는지 확인합니다. (필요시 `localStorage` 활용)
4.  **전환율 분석**: GA4(T-010)를 통해 각 단계별 이탈률을 추적하는 이벤트를 설정하고, 리디자인 전후의 전체 예약 완료율을 비교하여 개선 효과를 측정합니다.
