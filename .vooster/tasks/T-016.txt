# 새 디자인 가이드 기반 전체 랜딩페이지 UI/UX 리디자인

**Task ID:** T-016
**Status:** DONE
**Importance:** MUST
**Complexity:** 8/10
**Urgency:** 10/10
**Dependencies:** None

## Description

### 설명
새로운 디자인 가이드(주요 색상: #0054A6, #F0F7FF)를 기반으로 전체 랜딩페이지의 UI/UX를 전면 리디자인합니다. Mayo Clinic, Philips Healthcare와 같은 전문적이고 신뢰감 있는 스타일을 벤치마킹하여 공통 컴포넌트, 레이아웃, 시각적 요소를 재구성하고 브랜드 아이덴티티를 강화합니다.

### 구현 상세
1.  **디자인 시스템 설정 (Tailwind CSS)**: `tailwind.config.ts` 파일에 새로운 디자인 토큰을 정의합니다. 색상, 폰트, 간격, 그림자 등을 가이드에 맞춰 설정합니다.
    ```typescript
    // tailwind.config.ts
    module.exports = {
      theme: {
        extend: {
          colors: {
            primary: '#0054A6',
            'primary-light': '#F0F7FF',
            accent: '#00A8CC',
          },
          fontFamily: {
            sans: ['Pretendard', 'sans-serif'], // 예시 폰트
          },
        },
      },
      plugins: [],
    }
    ```
2.  **공통 컴포넌트 재구축 (shadcn/ui)**: `shadcn/ui`를 사용하여 버튼, 카드, 입력 필드, 모달, 뱃지 등 핵심 공통 컴포넌트를 새로운 디자인 가이드에 맞춰 재정의합니다. 기존 컴포넌트를 대체하거나 새로 생성하여 일관성을 확보합니다.
3.  **전역 스타일 및 레이아웃 재설계**: `app/globals.css` 파일에서 기본 배경색, 폰트 색상 등 전역 스타일을 업데이트합니다. `app/layout.tsx` 파일에서 헤더, 푸터, 메인 콘텐츠 영역 등 전체 페이지의 기본 레이아웃 구조를 새 디자인에 맞게 재구성합니다.
4.  **시각적 요소 적용**: 새로운 아이콘 시스템(예: `lucide-react`)을 도입하고, 카드형 디자인, CTA 버튼 스타일 등 새로운 시각적 요소를 페이지 전반에 일관되게 적용합니다.

### 테스트 전략
1.  **컴포넌트 단위 시각적 테스트**: Storybook을 사용하여 각 공통 컴포넌트가 디자인 가이드에 따라 정확하게 구현되었는지 확인합니다. 다양한 props에 따른 상태 변화(hover, focus, disabled 등)를 테스트합니다.
2.  **디자인-구현 비교 검증**: Figma 또는 제플린으로 제공된 디자인 시안과 실제 구현된 페이지(메인, 예약, FAQ 등)를 나란히 비교하며 픽셀 단위의 정확성, 색상, 여백 등이 일치하는지 검증합니다.
3.  **시각적 회귀 테스트**: Chromatic 또는 Percy와 같은 도구를 사용하여 UI 변경 전후의 스크린샷을 비교하고, 의도치 않은 시각적 깨짐이 발생하지 않았는지 자동으로 검사합니다.
